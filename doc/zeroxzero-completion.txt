1. INTRODUCTION                                 *zeroxzero-completion-intro*

zeroxzero-completion provides inline ghost text code suggestions in Neovim,
similar to GitHub Copilot or Supermaven. It routes completions through the
0x0 server, which calls the Anthropic API on your behalf.

Requires the zeroxzero plugin (|zeroxzero.txt|) to be installed and the
0x0 server to be running. Uses `claude-haiku-4-5-20251001` by default for
fast time-to-first-token (~200-400ms).

==============================================================================
2. SETUP                                        *zeroxzero-completion-setup*

Requirements:
- Neovim >= 0.10
- curl in PATH
- zeroxzero plugin installed and configured
- 0x0 server running (started by the zeroxzero plugin)
- `ANTHROPIC_API_KEY` env var set (or configured in 0x0 config)

Using lazy.nvim: >lua
  -- First, ensure zeroxzero is installed:
  { "anonymous-dev-org/0x0.nvim", opts = {} },

  -- Then add zeroxzero-completion:
  {
    "anonymous-dev-org/0x0-completion.nvim",
    dependencies = { "0x0.nvim" },
    event = "InsertEnter",
    opts = {},
  }
<

Manual setup: >lua
  -- After zeroxzero is configured:
  require("zeroxzero-completion").setup({
    -- options here
  })
<

==============================================================================
3. CONFIGURATION                               *zeroxzero-completion-config*

Default configuration: >lua
  {
    model = "claude-haiku-4-5-20251001",       -- model for completions
    max_tokens = 256,                          -- max completion length
    debounce_ms = 150,                         -- wait before requesting
    max_prefix_lines = 100,                    -- context before cursor
    max_suffix_lines = 50,                     -- context after cursor
    cache_size = 64,                           -- LRU cache entries
    disabled_filetypes = {                     -- skip these filetypes
      TelescopePrompt = true,
      NvimTree = true,
      lazy = true,
      mason = true,
      help = true,
      [""] = true,
    },
    keymaps = {
      accept = "<Tab>",                        -- accept full completion
      dismiss = "<C-]>",                       -- dismiss ghost text
      accept_word = "<M-w>",                   -- accept first word
      accept_line = "<M-l>",                   -- accept first line
      toggle = "<M-Bslash>",                   -- toggle on/off (normal mode)
    },
  }
<

API key configuration is handled by the 0x0 server. Set `ANTHROPIC_API_KEY`
in your environment, or configure it in your 0x0 config:
>yaml
  provider:
    anthropic:
      options:
        apiKey: "sk-ant-..."
<

==============================================================================
4. KEYMAPS                                    *zeroxzero-completion-keymaps*

Insert mode:
  <Tab>       Accept the full completion. Falls through to normal Tab
              behavior (indent, snippets) when no ghost text is visible.
  <C-]>       Dismiss the current ghost text.
  <M-w>       Accept the first word of the completion.
  <M-l>       Accept the first line of the completion.

Normal mode:
  <M-\>       Toggle completions on/off.

All keymaps are configurable via the `keymaps` option in setup().

==============================================================================
5. COMMANDS                                  *zeroxzero-completion-commands*

:ZeroCompletionToggle                        *:ZeroCompletionToggle*
  Toggle autocompletion on or off.

:ZeroCompletionClear                         *:ZeroCompletionClear*
  Clear the current ghost text.

==============================================================================
6. HEALTH CHECK                               *zeroxzero-completion-health*

Run `:checkhealth zeroxzero-completion` to verify your setup.

Checks:
- Neovim version >= 0.10
- curl available in PATH
- zeroxzero plugin installed
- 0x0 server reachable
- Current model and debounce settings
